# CP 前端用户快速上手

## 1. 一句话流程

先用 GUI 清洗 CP 数据，再点击 `CP Cockpit` 交互分析。

```text
选择公司
  -> 选择原始数据目录
  -> 选择输出目录
  -> 开始清洗
  -> 打开 CP Cockpit
  -> 查看良率、Bin、参数、Wafer、Cpk
```

## 2. 启动软件

在项目目录运行：

```powershell
python -m gui.multi_company_main
```

普通用户如果桌面已有快捷方式，双击快捷方式即可。

## 3. 清洗数据

1. 在左侧选择公司：
   - `HuaHong`
   - `JeTech`
   - `Lion`
   - `国宇FRD`
2. 在右侧输入或选择原始数据目录。
3. 输入或选择输出父目录。
4. 点击对应清洗或处理按钮。
5. 等待日志提示完成。

清洗完成后，输出目录中应生成：

```text
*_cleaned_*.csv
*_yield_*.csv
*_spec_*.csv
```

输入和输出文本框默认使用当前 Windows 用户桌面，例如：

```text
输入：C:\Users\<用户名>\Desktop
输出：C:\Users\<用户名>\Desktop
```

用户可直接在文本框中输入具体路径，也可以点击“选择文件夹”浏览选择。

## 4. 打开 CP Cockpit

点击当前公司页面中原“生成图表”位置的 `CP Cockpit` 按钮。左侧边栏只用于切换公司。

适合场景：

- 研发工程师想切换不同参数看分布。
- 质量工程师想查看 Bin、Cpk、超限。
- 工艺工程师想看 Wafer Map、区域差异。

打开后重点看这些页面：

| 页面 | 看什么 |
| --- | --- |
| 总览 | 总测试数、良率、Wafer 数、参数数 |
| 良率 / Bin | 哪些 Wafer 良率低，主要失效 Bin 是什么 |
| 参数 BoxPlot | 参数是否偏移，是否有离群点 |
| 参数 Wafer 散点图 | 每个参数在各 Wafer 上的原始点分布和离散程度 |
| Wafer Map | 失效或参数是否有空间分布 |
| 区域分析 | Center / Mid / Edge 是否有差异 |
| Wafer Summary | Wafer 间均值、标准差、中位数对比 |
| Cpk / 超限 | 哪些参数规格风险最大 |
| 数据预览 | 快速检查 CSV 内容 |

## 5. 推荐查看顺序

### 第一步：看良率

先看总良率和 Wafer 良率。

判断问题：

- 是否有单片 Wafer 明显低良率？
- 是否所有 Wafer 都低，可能是批次性问题？
- 是否只有某几片异常，可能是测试或局部工艺问题？

### 第二步：看失效 Bin

看失效 Bin Pareto。

判断问题：

- 最大失效 Bin 是哪个？
- 失效是否集中在一个测试项？
- 是否出现异常 Bin，和以往批次不同？

### 第三步：看关键参数 BoxPlot

选择研发或质量关注的关键参数。

判断问题：

- 中位数是否接近规格边界？
- 分布是否变宽？
- 是否有明显离群点？
- Wafer 之间是否差异很大？

### 第四步：看 Wafer Map 和区域

如果 cleaned CSV 有有效 `X/Y` 坐标，可查看 Wafer Map。

判断问题：

- 失效是否集中在边缘？
- 是否存在局部块状异常？
- Center / Mid / Edge 是否有系统性差异？

### 第五步：看 Cpk 和超限

质量或放行前重点查看。

判断问题：

- 哪些参数 Cpk 较低？
- 低于 LSL 或高于 USL 的数量是多少？
- 是否需要工程复判？

## 7. 如何理解常见现象

| 现象 | 可能原因 |
| --- | --- |
| 图表提示没有标准 CSV | 还未清洗，或输出目录选错 |
| Wafer Map 没有空间图 | cleaned CSV 的 `X/Y` 坐标为空或全为 0 |
| Cpk 没有结果 | spec CSV 缺规格，或参数数据不足 |
| 页面加载较慢 | BoxPlot 和散点页会展示所有参数，参数很多或数据量很大时图表数量较多 |
| 某个参数没有图 | 参数列不是数值，或全为空 |

## 8. 用户不要做的事

1. 不要手工改 cleaned CSV 中的测试值。
2. 不要把真实 CP 原始数据提交到 GitHub。
3. 不要把输出目录当成源代码目录提交。
4. 不要在前端里解释清洗错误；如果字段或单位不对，应回到清洗程序修正。

## 9. 给 IT 或开发人员的信息

如果用户反馈图表异常，请优先收集：

1. 原始数据来源公司。
2. 输出目录路径。
3. `cleaned/yield/spec` 三个 CSV 是否存在。
4. 异常页面截图。
5. 关键参数名和 Wafer ID。

然后按以下顺序排查：

```text
输出目录是否正确
  -> CSV 是否存在
  -> 字段是否符合 docs/data-contracts.md
  -> spec 上下限是否正确
  -> X/Y 是否有效
  -> 前端是否有加载错误
```
