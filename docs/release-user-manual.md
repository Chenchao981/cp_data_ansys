## 🚀 快速开始

### 环境要求

- Windows 10/11 (64-bit) 🪟
- **Anaconda3** (推荐) 🐍 或 Python 3.8+

### 三步上手

1. **解压** `release` 文件夹到任意位置（建议非系统盘📁）
2. **双击** `install_anaconda.bat`（首次运行必做✅）
   - 自动检测环境并安装 3 个必要包：`PyQt5`、`seaborn`、`openpyxl`
   - 全程自动化，无需人工输入⌨️
3. **双击** `start.bat` 运行程序
   - 出现主界面即表示安装成功 🎉

> ⚡ **提示**：第一次运行后可将 `start.bat` 固定到任务栏，启动更快捷！

## 🖱️ 界面操作

### 主界面布局 - 多公司版

```

国宇FRD目录支持：

- 输入单个批次目录。
- 输入一个产品目录，其第一层子目录为多个批次；每个批次下可包含一个或多个 EDS/数据子目录。
- 输出路径选择父目录，清洗时自动创建“第一个批次号_时间流水号”文件夹。
- 多批次合并输出时，每行保留原始 `Lot_ID`。
┌─────────────────────────────────────┐
│  🔬 CP数据分析工具 - 多公司版          │
├─────────────────────────────────────┤
│  📋 华虹(HH) │ 捷泰(JT) │ Lion │ 国宇FRD │  ← 公司选择导航栏
│  🎨 [亮色主题 / 暗黑主题]                 │
├─────────────────────────────────────┤
│  📁 数据来源: [选择数据源...]             │
│  📂 输出文件夹: [选择文件夹...]        │
├─────────────────────────────────────┤
│  🧹 [开始清洗数据]  📊 [生成图表]      │
├─────────────────────────────────────┤
│  📋 处理日志显示区域                  │
│                                     │
└─────────────────────────────────────┘
```

### 操作步骤

**可选：切换显示主题** 🎨

   - 默认使用暗黑主题。
   - 点击侧边栏底部的“亮色主题”按钮可切换到亮色界面；切换后按钮会显示“暗黑主题”。
   - 主题覆盖公司菜单、输入输出表单、操作按钮、日志、状态栏、数据源窗口和应用内弹窗，并会记住上次选择。

1. **选择公司类型** 🏢
   - 点击左侧导航栏中的华虹(HH)、捷泰(JT)、Lion 或国宇FRD按钮
   - 选中的公司按钮会高亮显示
   - 不同公司支持不同的数据格式：
     - **华虹(HH)**: 支持 `.txt`、`.dcp`
     - **捷泰(JT)**、**Lion**、**国宇FRD**: 支持 `.xls`、`.xlsx`
   - 四家公司都支持原始文件夹、单个 ZIP 和多个 ZIP

2. **选择数据来源** 📁
   - 点击“选择数据源...”，在同一个窗口中选择一个原始数据文件夹，或按 `Ctrl` / `Shift` 选择一个或多个 ZIP。
   - 选择窗口会自动区分文件夹和 ZIP；为避免重复处理，不允许“文件夹 + ZIP”混选或一次选择多个文件夹。
   - 也可以选择一个只包含 ZIP 的目录，后台会自动发现其中的 ZIP。
   - ZIP 会在后台临时目录中安全解压，处理完成后自动清理；原 ZIP 不会被修改。
   - 国宇 ZIP 会保留批次和 EDS 层级，可处理单批次 ZIP 或产品多批次 ZIP。

3. **选择输出文件夹** 📂
   - 点击输出路径的"选择文件夹..."
   - 或使用默认路径（桌面自动生成文件夹）
   - 此处选择的是父目录；四家公司都会自动创建“首个真实批次号_YYYYMMDD_HHMMSS”文件夹
   - 多批次输入按稳定的识别/处理顺序取第一个批次号；明细 CSV 仍保留全部批次的原始 `Lot_ID`
   - 同一秒内重复运行时追加 `_001`、`_002`，不会覆盖已有文件夹

4. **开始数据清洗** 🧹
   - 点击"开始清洗数据"按钮
   - 观察日志区域的处理进度
   - 等待"清洗完成"提示

5. **生成分析图表** 📊
   - 数据清洗完成后，"生成图表"按钮自动启用
   - 点击生成交互式 HTML 图表
   - 完成后会显示输出文件夹路径

## 📊 查看数据

### 输出文件位置

程序会在输出目录创建以下结构：

```
📂 批次号_YYYYMMDD_HHMMSS/
├── 📄 cleaned_data.csv          ← 清洗后数据
├── 📈 yield_chart_*.html        ← 良率图
├── 📦 boxplot_*.html           ← 箱体图  
├── 🔄 summary_boxplot.html     ← 汇总图
└── 📋 processing_log_*.log     ← 处理日志
```

### 查看方式

1. **CSV数据文件** 📄
   - 用 Excel 或记事本打开 `cleaned_data.csv`
   - 查看清洗后的结构化数据

2. **HTML图表文件** 📈
   - 双击 `.html` 文件用浏览器打开
   - 推荐使用 Chrome/Edge 浏览器
   - 支持交互操作：缩放、筛选、导出

3. **处理日志** 📋
   - 用记事本打开 `.log` 文件
   - 了解处理过程和异常信息

### 图表交互功能（重要）

- 🖱️ **鼠标悬停**：显示具体数值
- 🔍 **点击图例**：隐藏/显示数据系列  
- 📷 **图片下载**：右上角，最左边的按钮
- 🏠 **复位**：图表复位还原

## ❓ 常见问题

### 安装相关

**Q: 程序无法启动？** 🚫

- **A**: 检查是否已运行 `install_anaconda.bat`
- 命令行执行 `python --version` 确认环境 ✔️

**Q: 依赖安装失败？** ⚠️

- **A**: 重新运行 `install_anaconda.bat`
- 或手动执行：`pip install -r requirements_anaconda.txt` 🔧

### 使用相关

**Q: 找不到数据文件？** 🔍

- **A**: 根据选择的公司确保正确的文件格式：
  - **华虹(HH)**: 文件扩展名为 `.txt` 或 `.dcp`；ZIP 内也必须包含这些文件
  - **捷泰(JT)**、**Lion**、**国宇FRD**: 文件扩展名为 `.xls` 或 `.xlsx`
- 检查文件编码为 UTF-8 或 ANSI
- 确认文件未被占用或损坏
- 如果选择 ZIP，确认压缩包未加密、未损坏；不要在同一输入文件夹混放已解压原始文件和 ZIP，以免重复处理

**Q: 图表显示异常？** 📊

- **A**: 使用 Chrome/Edge 浏览器打开 HTML
- 检查数据是否包含有效数值
- 查看日志文件了解详细错误

**Q: 界面出现乱码？** 🌐

- **A**: `start.bat` 已自动设置 UTF-8 编码
- 如仍有问题，手动在命令行执行：`chcp 65001`

### 性能相关

**Q: 处理速度慢？** ⏱️

- **A**: 将数据放在本地硬盘（非网络驱动器）
- 关闭其他占用资源的程序
- 分批处理超大数据集（建议<1000文件）

**Q: 内存不足？** 💾

- **A**: 处理大数据集时关闭其他程序
- 分批处理，避免一次加载过多文件
- 建议系统内存 ≥ 8GB
