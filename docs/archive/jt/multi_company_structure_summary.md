## 🎯 核心设计特点

### ✅ 模块化隔离
- **独立命名空间**: `company_adapters` 作为独立模块，避免与现有代码冲突
- **清晰职责**: 专注于厂商数据格式适配转换
- **接口标准化**: 通过基类定义统一的适配器接口

### ✅ 可扩展架构  
- **基类设计**: `BaseCompanyAdapter` 定义抽象方法和通用功能
- **配置驱动**: `company_config.py` 集中管理厂商差异配置
- **插件化**: 新增厂商只需继承基类并添加配置

### ✅ 完整测试支持
- **单元测试**: 每个适配器对应独立测试文件
- **样本数据**: 分厂商的测试数据目录结构
- **自动化**: 支持 pytest 测试框架

## 🚀 核心组件功能

### 1. BaseCompanyAdapter (适配器基类)
- **抽象接口**: `transform_to_standard_format()`, `get_field_mapping()`
- **通用功能**: 数据验证、字段映射、单位转换
- **日志支持**: 详细的处理过程日志记录

### 2. CompanyConfig (配置管理器)
- **厂商配置**: HH公司（标准格式）、JT公司（待完善）配置
- **自动识别**: 基于文件路径和文件名的厂商识别
- **配置验证**: 配置完整性检查和更新功能

### 3. HHAdapter (HH适配器实现)
- **标准格式**: 作为基准格式的参考实现
- **数据验证**: HH格式特定的数据完整性检查
- **开发模板**: 为其他厂商适配器提供实现参考

## 🔄 数据转换流程设计

```
📁 原始数据 → 🔍 厂商识别 → 📖 专用读取器 → 🔄 厂商适配器 → 📊 统一CPLot → 📋 标准CSV → 📈 图表生成
     ↓              ↓              ↓              ↓              ↓           ↓
   JT格式        自动识别        JTReader       JTAdapter       CPLot     HH格式CSV
   HH格式        路径识别        DCPReader      HHAdapter       CPLot     HH格式CSV
```

## 📊 技术实现亮点

### 🎯 配置驱动设计
```python
COMPANY_CONFIGS = {
    'HH': {
        'field_mapping': {'Lot_ID': 'Lot_ID'},    # 恒等映射
        'unit_conversion': {}                      # 无需转换
    },
    'JT': {  
        'field_mapping': {'jt_lot_id': 'Lot_ID'}, # JT→HH映射
        'unit_conversion': {'VTH': {'factor': 0.001}} # mV→V转换
    }
}
```

### 🔌 插件化架构
```python
# 添加新厂商只需3步：
# 1. 配置：在company_config.py中添加厂商配置
# 2. 适配器：继承BaseCompanyAdapter实现转换逻辑  
# 3. 测试：添加对应的单元测试文件
```

### 🛡️ 数据验证机制
- **格式验证**: 检查CPLot对象数据完整性
- **字段验证**: 确保必需字段存在
- **数值验证**: Bin值范围检查
- **单位验证**: 转换前后数据一致性
