# 🏗️ CP数据分析平台模块化重构计划

## 📋 项目背景

当前项目已实现华虹(HH)和Jetech(JT)两家公司的数据清洗和图表生成功能。为支持后续引入DJ、CD等8家公司的数据分析，需要进行模块化重构，实现"一次开发，通用复用"的架构。

## 🎯 重构目标

1. **标准化数据接口** - 所有公司数据清洗后输出统一的3个CSV文件
2. **模块化架构** - 新增公司仅需实现数据清洗，其他功能自动复用
3. **零侵入集成** - 现有图表生成、GUI、打包模块保持不变
4. **可扩展性** - 支持快速添加新公司数据源

## 🏗️ 模块化架构设计

### 架构概览
```
原始数据 → 公司识别 → 公司适配器 → 标准数据处理 → 统一CSV输出 → 图表生成 → GUI展示
```

## 🔄 数据流标准化

### 1. 输入层
```
各公司原始数据 → 公司识别 → 格式读取器 → 原始CPLot对象
```

### 2. 适配层
```
原始CPLot对象 → 公司适配器 → 字段映射 → 单位转换 → 标准CPLot对象
```

### 3. 处理层
```
标准CPLot对象 → 标准处理器 → 数据验证 → 标准CSV文件
```

### 4. 输出层
```
标准CSV文件 → 图表生成器 → HTML图表 → GUI展示
```
